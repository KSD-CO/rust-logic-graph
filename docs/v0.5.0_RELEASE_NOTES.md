# Release Notes - v0.5.0

**Release Date**: November 16, 2025
**Codename**: Performance & Developer Tools Release

## üéØ Overview

Version 0.5.0 is a major release focused on **performance improvements** and **developer experience**. This release upgrades the core rule engine to use the RETE-UL algorithm (2-24x faster), adds comprehensive CLI developer tools, and introduces a high-performance caching layer.

## üöÄ Major Features

### 1. RETE-UL Algorithm (2-24x Performance Boost)

**Upgraded rust-rule-engine from v0.10 ‚Üí v0.14.0**

- **Performance**: 2-24x faster rule matching
- **Algorithm**: RETE-UL (RETE with Unlinking) for better memory efficiency
- **Compatibility**: 98% Drools compatible (up from 97%)
- **Migration**: 100% backward compatible API - no code changes required

**Performance Comparison:**

| Rule Complexity | v0.10 (RETE) | v0.14 (RETE-UL) | Speedup |
|----------------|--------------|------------------|---------|
| Simple (< 10 conditions) | 100ms | 50ms | 2x |
| Medium (10-50 conditions) | 500ms | 50ms | 10x |
| Complex (50+ conditions) | 2000ms | 80ms | 24x |
| Memory Usage | Baseline | -30% | More efficient |

**See**: [Migration Guide](MIGRATION_GUIDE.md)

### 2. CLI Developer Tools (`rlg`)

A comprehensive command-line tool for graph development and debugging.

#### Commands

**Validate**
```bash
rlg validate --file graph.json --verbose
```
- Checks node uniqueness
- Validates edge references
- Detects cycles
- Finds unreachable nodes
- Comprehensive error reporting

**Visualize**
```bash
rlg visualize --file graph.json --details
```
- ASCII tree visualization
- Node and edge listings
- Statistics (entry points, exit points)
- Graph structure analysis

**Profile**
```bash
rlg profile --file graph.json --iterations 100
```
- Performance profiling
- Min/max/average execution times
- Throughput calculation
- Multi-iteration benchmarking

**Dry-Run**
```bash
rlg dry-run --file graph.json --context data.json --verbose
```
- Execution plan preview
- Topological order display
- No actual execution
- Context loading support

**See**: [CLI Tool Guide](CLI_TOOL.md)

---

### 3. High-Performance Caching Layer

Enterprise-grade caching with multiple eviction policies.

**Features:**
- **TTL-based expiration** - Automatic cache invalidation
- **Eviction policies** - LRU, LFU, FIFO
- **Memory limits** - Configurable max entries
- **Statistics** - Hit rates, eviction counts
- **Cache invalidation** - Manual cache clearing

**Usage:**
```rust
use rust_logic_graph::{CacheManager, CacheConfig, EvictionPolicy};

let config = CacheConfig {
    max_entries: 1000,
    default_ttl: Some(Duration::from_secs(3600)),
    eviction_policy: EvictionPolicy::LRU,
};

let cache = CacheManager::new(config);
```

**See**: [Cache Implementation Guide](CACHE_IMPLEMENTATION.md)

## üîÑ Migration Guide

### Upgrading from v0.10.x

**Step 1: Update Cargo.toml**
```toml
# Before
rust-rule-engine = "0.10"

# After
rust-rule-engine = "0.14.0"
```

**Step 2: Rebuild**
```bash
cargo clean
cargo build --release
```

**Step 3: Test**
```bash
cargo test
```

**That's it!** No code changes required.
